# v2.0.10 — Trading riparato (markup NaN) + Venditore Ambulante reale

**Data**: 2026-07-12
**Contesto**: blocco 3 della campagna di chiusura. Il baratto era **completamente morto**: nessun mercante in `traders.json` definiva `baseMarkup`, quindi `calculateEffectiveMarkup(undefined, …)` produceva NaN, che propagava a ogni prezzo e al `balance` — il gate `balance >= 0` era sempre falso, nessuno scambio finalizzabile ("Markup: NaN%").

## Fix
- **`public/data/traders.json`** (+ sync `src/data`): `baseMarkup: 1.3` per Marcus; aggiunto il mercante **`wandering_trader`** ("Venditore Ambulante", markup 1.5, inventario con soli item id verificati: CONS_001/002, MED_BANDAGE_BASIC, MED_ANTIDOTE, scrap_metal, ammo_9mm). Era referenziato da `unique_wandering_trader_encounter` ma non esisteva → "Trader not found".
- **`tradingService.startTradingSession`**: guardia su `baseMarkup` mancante/non finito → default 1.3 con warn. Il trading non può più andare in NaN per dati incompleti.
- **`eventStore`**: l'effetto placeholder `trade_screen_placeholder` dell'incontro casuale `random_wandering_trader` ora apre la vera sessione di scambio col Venditore Ambulante (+ testo journal aggiornato in `encounters.json`).

## Verifica
- `tsc` pulito sui file toccati, `npm run build` ✅, JSON validati.

## Nota
Il campo `price` per-item dei mercanti resta dead-data (il baratto usa `item.value` dal DB): comportamento invariato, nessun impatto.
