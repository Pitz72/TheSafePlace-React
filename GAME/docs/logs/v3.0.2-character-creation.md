# v3.0.2 — CharacterCreationScreen redesign

**Data:** 2026-05-15
**Tipo:** UI redesign + meccanica (screen)

---

## Modifiche

### CharacterCreationScreen.tsx — riscritto da zero

**Concept cambiato rispetto al v2:**
Il vecchio screen mostrava un'animazione di roll 4d6 sequenziale (CRT style).
Il GDD v3 stabilisce che Ultimo ha stat fisse e la sola bussola morale è configurabile
via 3 domande narrative. Il roll dei dadi non esiste più.

**Nuovo design — "Open Book" dal handoff:**
- Layout libro aperto a due pagine su sfondo scuro
- Binding centrale con ombra e linea
- Classe `.paper` con texture carta su entrambe le pagine

**Pagina sinistra — Identità fissa:**
- Intestazione "Scheda Sopravvissuto" + timbro "FISSO" (rust, ruotato)
- Nome "Ultimo" in italic EB Garamond + annotazione manoscritta
- Portrait sketch SVG (silhouette su sfondo scuro)
- Aspetto e età (16 anni) in Caveat
- Stats 2×3 grid: FOR 9, DES 12, COS 10, INT 13, SAG 11, CAR 11 (fisse per GDD v3)
  con modificatori colorati (mint positivo, rust negativo)

**Pagina destra — Bussola morale:**
- 3 domande narrative da GDD v3 con 2 opzioni ciascuna
- Selezione via click, stile radio con checkbox ink-style
- Barra visuale Lena/Elian con ago mobile (transizione CSS 0.4s)
- Testo dinamico sotto la barra ("le tue scelte lo decideranno" / "le tue scelte hanno parlato")
- Bottone "COMINCIA IL VIAGGIO" disabilitato (opacity 35%) finché non tutte e 3 risposte

**Logica:**
- `setAttributes({ for: 9, des: 12, cos: 10, int: 13, sag: 11, car: 11 })` — stat fisse
- `useCharacterStore.setState({ alignment: ... })` — scrittura diretta, bypassa i trigger
  di soglia di `changeAlignment` (che sparerebbe journal entries durante l'inizializzazione)
- `setGameState(GameState.IN_GAME)` al conferma
- `setGameState(GameState.MAIN_MENU)` al back / Escape
- Enter avvia il gioco se tutte le domande sono risposte
