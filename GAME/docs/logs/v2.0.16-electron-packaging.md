# v2.0.16 — Packaging desktop Electron (Win/Mac/Linux)

**Data**: 2026-07-12
**Contesto**: chiusura del progetto come app desktop. Scelto **Electron** (più immediato di Tauri, nessuna toolchain Rust). Nessuna modifica al codice di gioco: è puro packaging del bundle Vite.

## Componenti aggiunti
- **`electron/main.cjs`** — processo main. Il gioco carica i dati con `fetch('data/...')`, che Chromium blocca su `file://`; quindi il `dist/` viene servito su uno **schema custom privilegiato `app://`** (index a `app://bundle/index.html`). Lo schema è `standard + secure` → il renderer ha un'origine stabile e **localStorage (i salvataggi) persiste** tra gli avvii. File serviti via `fs.readFile` (asar-transparent), con MIME corretto e fallback SPA.
  - Link http(s) esterni → browser di sistema. Menu bar nascosta. Finestra 1280×800 (min 1024×640), fondo nero, icona custom.
  - Modalità dev: `electron . --dev` carica `http://localhost:3000` (Vite) + DevTools.
- **`build/icon.png`** (1024×1024) — icona app generata dal logo Runtime Multimedia su fondo nero. electron-builder ne deriva `.ico`/`.icns`.
- **`package.json`** — `main: electron/main.cjs`, script (`electron`, `electron:dev`, `dist`, `dist:win|mac|linux`), config `build` electron-builder:
  - Win: `nsis` (installer) + `portable`
  - Mac: `dmg` + `zip` (categoria role-playing-games)
  - Linux: `AppImage` + `deb`
  - `files` limitato a `dist/ + electron/ + package.json` (le dipendenze React sono già bundlate da Vite → nessun node_modules nel pacchetto).
- **`.github/workflows/build-desktop.yml`** — matrix `windows/macos/ubuntu-latest`. Trigger: manuale (`workflow_dispatch`) o push di un tag `v*`. Ogni runner builda i target del proprio OS (`npm ci` → `npm run build` → `electron-builder`) e carica gli installer come artifact.

## Verifica locale (Windows)
- `electron-builder --dir --win`: pacchetto unpacked generato, asar + firma OK.
- Smoke test: `The Safe Place.exe` avviato — 4 processi Electron vivi dopo 8s, **nessun crash di boot** (main process e schema `app://` caricano correttamente).

## Come produrre gli installer
- **Locale** (solo OS corrente): `npm run dist` in `GAME/` → output in `GAME/release/`.
- **Tutti gli OS**: push di un tag `v*` (o "Run workflow" da Actions) → installer Win/Mac/Linux come artifact scaricabili.

## Note
- Il font retro VT323 è caricato da Google Fonts (rete): offline il gioco ripiega su monospace di sistema. Bundlarlo in locale è un possibile ritocco futuro, non bloccante.
- macOS: gli artifact non sono firmati/notarizzati (avviso Gatekeeper al primo avvio: click destro → Apri). La firma richiede un Apple Developer ID, fuori scope.
