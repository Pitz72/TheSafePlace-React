# v3.0.4 — EventScreen redesign (lettera di carta)

**Data:** 2026-06-11
**Tipo:** UI redesign (overlay eventi)

---

## Decisione di design

Il handoff proponeva 2 varianti: A (backdrop atmosferico + lettera) e
B (solo lettera centrata, per cutscene/memorie). Implementata la **A**
con backdrop generico (gradiente scuro + polvere + grana) invece della
scena-foresta specifica del mockup: il backdrop per-evento/per-bioma
potrà arrivare quando ci saranno gli asset.

## Modifiche

### EventScreen.tsx — riscritto da zero

**Cosa resta del v2 (logica invariata):**
- Dual-mode: eventi legacy via `eventStore`, cutscene Ink via
  `narrativeStore` (`isStoryActive`) con `narrativeService.chooseChoiceIndex`
- Sostituzione `{RANDOM_DONOR}`, requisiti oggetto (`itemRequirements`),
  schermata di esito (`eventResolutionText`), suoni navigate/confirm/error

**Nuova resa visiva:**
- Backdrop scuro atmosferico, label in alto "EVENTO · LUOGO" (da
  `getTileInfo`) + motto in Caveat
- Lettera di carta (`.paper`, ruotata −0.8°, Tape) ancorata in basso:
  header titolo + "GIORNO N · HH:MM", narrazione in EB Garamond
  (primo paragrafo pieno, successivi corsivi sbiaditi — split su `\n`),
  scrollabile se lunga
- **Scelte numerate con tag meccanico visibile** (GDD v3, Sistema 2):
  - skill check → `SKILL DC X` (slate)
  - allineamento Lena → `COMPASSIONE` (mint), Elian → `PRAGMATISMO` (rust)
  - requisito oggetto mancante → `RICHIEDE …` (rust) + scelta disabilitata
- **Input misto**: click conferma direttamente, hover seleziona,
  1–9 selezionano, ↑↓/W/S navigano, Invio conferma
- Esito: stessa lettera con label "ESITO" e bottone "CONTINUA IL VIAGGIO"

## Limiti noti

- Il tag meccanico per le scelte **Ink** non esiste ancora (le choice Ink
  non trasportano metadati skill/DC) — si potrà fare con tag Ink
  `#tag:...` quando servirà.
- Backdrop unico per tutti i biomi.
