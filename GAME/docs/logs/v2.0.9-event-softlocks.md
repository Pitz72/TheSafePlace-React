# v2.0.9 — Soft-lock eventi eliminati + schema dati tollerante

**Data**: 2026-07-12
**Contesto**: blocco 2 della campagna di chiusura. Tre eventi unici raggiungibili in una partita normale congelavano il gioco: la scelta produceva un esito vuoto, la schermata di risoluzione (gated su stringa truthy) non appariva mai, e la EventScreen non aveva ESC → soft-lock totale.

## Cause radice (in `eventStore.resolveEventChoice`)
1. **`unique_hermit_cabin`** (Foresta): outcome con `type: "special"` a livello *outcome* — il loop gestiva solo `direct`/`skillCheck` e lo ignorava in silenzio (il dialogo con l'eremita non partiva nemmeno).
2. **`unique_titanic_car`** (Pianura): `journalEntry` con testo annidato in `value.text`, ma il codice leggeva solo `result.text`.
3. **`unique_dogbuster_drone`** (Città): `alignmentChange` scartava d'ufficio il testo narrativo (`message = null`).

## Fix engine-side (`src/store/eventStore.ts`)
- Loop outcome: nuovo ramo `special` a livello outcome → instradato in `applyResult` (il dialogo dell'eremita ora parte davvero) + eventuali `results` extra processati.
- `journalEntry`: legge `result.text ?? result.value?.text`.
- `alignmentChange`: il testo narrativo autoriale finisce nell'esito (le voci di journal sull'allineamento restano a `changeAlignment`).
- **Rete di sicurezza**: se `resolutionSummary` è vuoto → fallback `"Prosegui per la tua strada."`. La schermata di esito appare SEMPRE: il soft-lock da esito vuoto è impossibile anche per errori dati futuri.
- **Schema item tollerante** (`readItemRef`): `addItem`/`removeItem` accettano sia `{value:{itemId,quantity}}` sia `{value:"<itemId>", quantity?}` (schema usato da tutto `unique_donor_events.json` — 8 ricompense che prima diventavano `addItem(undefined)` e testo narrativo perso). Il testo autoriale va nell'esito, il journal riceve la riga item pulita.

## Fix UI (`src/components/EventScreen.tsx`)
- **ESC hatch**: in schermata esito ESC = INVIO (dismiss); in schermata scelte ESC funziona SOLO se nessuna scelta è selezionabile (tutte gated da requisiti item) → "vai oltre" con suono cancel. Footer dinamico `[ESC] Vai oltre` in quel caso.

## Verifica
- `npx tsc --noEmit`: nessun errore nei file toccati. `npm run build` ✅.

## Nota
La parte "schema donor addItem" del Blocco 4 è risolta qui engine-side (nessuna modifica ai 10 file dati necessaria). Restano per il Blocco 4: caricamento `repair_kits.json`/`unique_items.json` + id rotti nelle ricompense quest.
