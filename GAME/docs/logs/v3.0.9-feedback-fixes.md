## Feedback ricevuto e fix

### 1. "Togli boot e ringraziamenti all'inizio"
- `gameStore.gameState` iniziale: `INITIAL_BLACK_SCREEN` → `MAIN_MENU`.
  Il gioco parte direttamente sul menu. `BootScreen` e i 4 GameState di
  boot restano nel codice ma sono irraggiungibili (da rimuovere in un
  cleanup futuro).

### 2. "Font tutti molto piccoli"
Scala tipografica alzata globalmente (~+25/30%):
- `diary.css`: `.t-label` 11→14px, `.t-label-sm` 9→12px, `.tsp-btn` 13→15px
- Componenti ui: `JournalEntry` (serif 14→16, hand 17→20, ora 10→12),
  `StatMeter` (etichette 10→12, valori 12→14, colonne più larghe)
- Tutte le schermate v3: GameScreen (top bar 22→27, hand 14→17, ecc.),
  EventScreen (prosa 18→20, scelte 15→17, tag 9→11), DialogueScreen
  (nome 28→32, battuta 16→19, risposte 14→16), CutsceneScreen (prosa
  17→20, scelte 16→18), GameOverScreen (prosa 17→19, ricordi 15→18),
  CharacterCreationScreen (domande 15→18, opzioni 14→16, stat 20→22)
- Le card laterali della GameScreen allargate (240→280, 320→380) per
  i font maggiori

### 3. "Scritte bianche sopra i pezzi di carta = casino"
I testi chiari della GameScreen stanno sopra la mappa, che è carta
chiara — dove il gradiente era debole risultavano illeggibili:
- Gradienti top/bottom bar molto più solidi (0.96 → trasparente, più alti)
- `text-shadow: 0 1px 3-4px rgba(10,13,18,0.9)` su top bar, action bar,
  label EVENTO della EventScreen e hint "ROTELLA ZOOM" della mappa

### 4. "La mappa appare piccola al centro"
La WorldMap partiva a zoom 1 (1200×800 px fissi) lasciando bordo scuro:
- **Zoom cover calcolato a runtime** (ResizeObserver): la mappa riempie
  sempre il contenitore; zoom iniziale = cover × 1.15
- **Zoom minimo = cover**: non si può rimpicciolire sotto il full-bleed
- **Pan vincolato** dentro i limiti del mondo: mai bordi scuri visibili
- Etichette mappa più grandi (nomi 15→18, note 11→13, "tu sei qui" 12→15)

### 5. "Non è Electron? Anteprima full screen senza browser?"
Il progetto è **React + Tauri** (deciso, non Electron) ma Tauri non è
ancora integrato — è il passo pianificato dopo il completamento UI.
Nel frattempo: **`npm run app`** (nuovo script) apre il gioco in una
finestra Edge dedicata in modalità app, fullscreen, senza barra del
browser. Richiede `npm run dev` attivo in un altro terminale.
