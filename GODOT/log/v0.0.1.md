# Changelog v0.0.1 - Genesi del Progetto

## Panoramica
Questa versione segna la nascita ufficiale del porting di "The Safe Place" su motore Godot 4.x. L'obiettivo principale di questa fase è stato stabilire fondamenta solide e scalabili, migrando la logica e i dati dalla versione React originale in un'architettura nativa per Godot. Abbiamo posto particolare enfasi sulla modularità e sulla gestione dei dati, assicurando che l'enorme mole di contenuti (dialoghi, oggetti, quest) fosse immediatamente accessibile nel nuovo ambiente.

## Dettagli Tecnici

### 1. Architettura del Progetto
Abbiamo definito una struttura di directory pulita e funzionale per facilitare lo sviluppo futuro:
*   `_core/`: Contiene i "cervelli" del gioco, script Singleton che gestiscono lo stato globale.
*   `assets/`: Repository per gli asset grezzi (grafica, audio).
*   `data/`: Il cuore pulsante dei dati, contenente i file JSON e le definizioni delle Risorse.
*   `scenes/`: Organizzazione logica delle scene di gioco (UI, Mondo, Entità).
*   `scripts/`: Libreria di script di utilità e logica condivisa.

### 2. Core Systems (Autoloads)
Sono stati implementati i seguenti sistemi centrali, registrati come Autoload per essere accessibili ovunque:
*   **GameManager (`GameManager.gd`)**: Gestisce lo stato globale del gioco (Menu, In-Game, ecc.), la posizione del giocatore e i flag di progresso.
*   **EventBus (`EventBus.gd`)**: Un sistema di segnali centralizzato che disaccoppia i vari componenti del gioco. L'UI non deve conoscere il Player, ascolta solo gli eventi.
*   **TimeSystem (`TimeSystem.gd`)**: Gestisce il ciclo giorno/notte e il passaggio del tempo, fondamentale per le meccaniche survival.
*   **DataLoader (`DataLoader.gd`)**: Un sistema robusto per importare e parsare i file JSON originali (Items, Enemies, Quests, Dialogues) al lancio del gioco, popolando dizionari globali.
*   **AudioManager (`AudioManager.gd`)**: Un placeholder strutturato per il futuro sistema audio procedurale.

### 3. Data Layer
Per sfruttare la potenza di Godot, abbiamo iniziato a mappare i dati JSON in strutture tipizzate:
*   Creati script `Resource` personalizzati: `ItemData`, `EnemyData`, `QuestData`. Questo permetterà in futuro di editare questi dati direttamente nell'Inspector di Godot se necessario.
*   Migrazione completa dei file JSON dalla versione React alla cartella `GODOT/data/json`.

### 4. World Building
*   **MapGenerator (`MapGenerator.gd`)**: Implementato un algoritmo che legge la stringa ASCII della mappa (definita nel GDD) e la prepara per essere convertita in una TileMap.
*   **Tileset**: Creato e importato `tileset.svg`, una rappresentazione vettoriale dei tile in stile Ultima/C64.

### 5. Interfaccia Utente (UI)
*   **HUD (`HUD.tscn`)**: Creata la scena base per l'interfaccia di gioco, con barre per salute/stamina e un pannello log per i messaggi.
*   **Journal (`Journal.tscn`)**: Implementata la struttura per il diario di gioco, con tab per Quest, Inventario e Lore.
*   **Input Map**: Configurati gli input di base (WASD per il movimento, E per interagire) nel `project.godot`.

## Prossimi Passi
Il sistema è ora pronto per la logica di gioco vera e propria. I prossimi passi si concentreranno sul collegare questi sistemi: far muovere il giocatore sulla mappa generata, visualizzare i dati dell'inventario nella UI e attivare i primi eventi di gioco.
